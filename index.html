Phase I · IA Blueprint

Structure Before Code

Information Architecture is the art of organizing for findability. Every line of HTML is a design decision.

IA

Information Architecture

Define the sitemap before touching an editor. Logical hierarchy means content is findable and usable — not just present.

SEM

Semantic DOM

Replace div soup with meaningful landmarks: <header>, <main>, <article>, <footer>. Machines and humans both benefit.

DRY

DRY CSS via BEM

Define a component once, reuse everywhere. Block__Element--Modifier naming removes ambiguity and eliminates style duplication.

A11Y

Accessibility First

Every image carries an alt description. Contrast ratios exceed 4.5:1. The web must be perceivable by all users and assistive technologies.

Core Skills · Project 1

The Engineering Stack

Each discipline is a layer in the final render. Master each layer before moving to the next.

HTML Fundamentals

Semantic markup transforms raw content into a navigable, machine-readable structure. One <h1>. Proper landmarks. Zero div soup.

Core Skill

CSS Layout Systems

CSS Grid owns the macro page structure. Flexbox handles micro component alignment. Each tool has its domain — never swap them.

Core Skill

Quality Gate

Code does not ship until it passes W3C validation, a semantic audit, and a pixel-perfect visual regression check. Zero errors allowed.

Validation

CSS Engineering Principles

Separation of concerns: HTML owns structure, CSS owns appearance. No inline styles — ever. One external stylesheet, shared across all pages. A component defined once is a component maintainable everywhere.

Architecture

The IPO Engineering Mindset

Input (assets: images, icons, text) → Process (CSS box model, logic) → Output (pixel-perfect render). Debug by isolating the stage that failed. Every layout issue has an address.

Mindset
